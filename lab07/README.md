# Марковская модель погоды

## Цель работы

Разработать графическое приложение для имитационного моделирования смены погодных состояний на основе цепи Маркова с непрерывным временем, провести статистическую обработку результатов и сравнить эмпирическое распределение времени пребывания в состояниях с теоретическим стационарным распределением.

## Постановка задачи

Погода моделируется как случайный процесс с тремя состояниями:

| Состояние | Название |
|-----------|----------|
| 1 | Ясно |
| 2 | Облачно |
| 3 | Пасмурно |

Единица времени - **1 день**. Процесс описывается цепью Маркова с непрерывным временем, заданной матрицей интенсивностей $Q$.

## Математическая модель

### Матрица интенсивностей

Матрица $Q$ задаётся условиями:

- внедиагональные элементы $q_{ij} \ge 0$ - интенсивности переходов из состояния $i$ в $j$;
- диагональные элементы $q_{ii} = -\sum_{j \ne i} q_{ij} < 0$;
- сумма элементов каждой строки равна нулю: $\sum_{j} q_{ij} = 0$.

В эксперименте использовалась следующая матрица:

$$Q = \begin{pmatrix} -0.20 & 0.15 & 0.05 \\ 0.10 & -0.25 & 0.15 \\ 0.05 & 0.10 & -0.15 \end{pmatrix}$$

### Моделирование методом следующего события

Время пребывания процесса в состоянии $i$ распределено показательно с параметром $\lambda_i = -q_{ii}$:

$$\tau_i \sim \text{Exp}(\lambda_i), \qquad \mathbb{E}[\tau_i] = \frac{1}{\lambda_i}$$

Генерация времени до следующего перехода:

$$\tau = -\frac{\ln \alpha}{\lambda_i}, \quad \alpha \sim U(0, 1)$$

Следующее состояние $j \ne i$ выбирается с вероятностью:

$$p_{ij} = \frac{q_{ij}}{\lambda_i}$$

### Стационарное распределение

Теоретическое стационарное распределение $\pi$ находится как решение системы:

$$\pi Q = 0, \qquad \sum_{i} \pi_i = 1$$

В матричном виде заменяем последнее уравнение условием нормировки и решаем:

$$A^{\top} \pi = b, \quad \text{где } A = Q,\ b_i = 0,\ b_n = 1,\ A_{n,\cdot} = \mathbf{1}$$

Для заданной матрицы $Q$ теоретическое стационарное распределение:

| Состояние | $\pi_i$ (теор.) |
|-----------|----------------|
| Ясно | 0.3846 |
| Облачно | 0.3846 |
| Пасмурно | 0.2308 |

## Статистическая обработка результатов

По итогам симуляции $T$ дней фиксируется суммарное время $t_i$, проведённое в каждом состоянии. Эмпирическая доля времени:

$$\hat{\pi}_i = \frac{t_i}{\sum_{j} t_j}$$

Отклонение от теоретического значения:

$$\delta_i = \hat{\pi}_i - \pi_i$$

По закону больших чисел при $T \to \infty$:

$$\hat{\pi}_i \xrightarrow{} \pi_i$$

## Описание программы

Приложение реализовано на Python с использованием `tkinter` (GUI), `matplotlib` (графики) и `numpy` (линейная алгебра). Моделирование выполняется в отдельном потоке, графики обновляются в реальном времени.

**Элементы управления:**

- Матрица интенсивностей $Q$ - задаётся вручную, предусмотрена автоматическая пересчёт диагонали;
- Число дней симуляции, задержка обновления, частота перерисовки;
- Кнопки: Запуск / Стоп / Продолжить / Сохранить в CSV.

**Визуализация:**

- Хронология смены состояний - ступенчатый график с цветовой заливкой;
- Сравнение эмпирических и теоретических долей - сгруппированные столбцы;
- Круговая диаграмма эмпирических долей времени.

**Сохранение результатов:**

Лог переходов сохраняется в `weather_log.csv` со столбцами:

| Поле | Описание |
|------|----------|
| `transition_id` | Номер перехода |
| `time_start` / `time_end` | Начало и конец пребывания (дни) |
| `duration_days` | Длительность пребывания |
| `state_id` / `state_name` | Состояние |
| `timestamp_real` | Реальное время записи |

Сводная статистика сохраняется в `weather_log_summary.csv` с эмпирическими и теоретическими вероятностями и отклонениями $\delta_i$.

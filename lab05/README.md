## Моделирование случайных событий (GUI)

## Цель работы

Разработать графическое приложение для моделирования случайных событий с использованием генератора случайных чисел и дискретных вероятностных распределений.

## Приложение «Скажи “да” или “нет”»

Для моделирования используется базовый датчик случайных чисел $\alpha$, равномерно распределённых на интервале $[0,1]$.

Рассматривается событие $A$ — ответ **«ДА»**.

Вероятность события:
$P(A) = 0.5$

Событие $A$ считается наступившим, если выполняется условие:
$\alpha < 0.5$

В противном случае реализуется противоположное событие — ответ **«НЕТ»**.

Таким образом, реализуется схема Бернулли с равновероятными исходами.

## Приложение «Шар предсказаний» (Magic 8-Ball)

Рассматривается полная группа попарно несовместных событий:
$A_1, A_2, \dots, A_{20}$

Каждому событию соответствует определённый ответ шара:
- Это бесспорно
- Определенно да
- Знаки указывают, что да
- Мои источники говорят да
- ДА!
- Перспектива хорошая
- Да
- Наиболее вероятно
- Определенно нет
- Я так не думаю
- Знаки указывают, что нет
- Мои источники говорят нет
- НЕТ!
- Извини, нет
- Перспектива не очень хорошая
- Очень сомневаюсь
- Не могу сейчас сказать
- Будущее туманно, спроси позже
- Соберись с мыслями и спроси снова
- Лучше тебе не говорить сейчас

Вероятности событий одинаковы:
$p_i = 0.05, \quad i = 1, \dots, 20$

Сумма вероятностей:
$\sum_{i=1}^{20} p_i = 1$

### Метод моделирования

Для выбора события используется случайное число $\alpha \in [0,1]$.

Единичный отрезок $[0,1]$ разбивается на интервалы, длины которых равны вероятностям $p_i$.

Выбирается событие $A_k$, для которого выполняется:
$\sum_{i=1}^{k-1} p_i < \alpha \le \sum_{i=1}^{k} p_i$

## Сбор статистики

В процессе моделирования для каждого события вычисляются:
- $n_k$ — число реализаций события  
- $p_k = \frac{n_k}{N}$ — относительная частота  

где:
- $N$ — общее число испытаний
